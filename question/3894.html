Как работает priority_queue

`priority_queue` является контейнером-адаптером, который обеспечивает автоматическое упорядочивание своих элементов по приоритету. Элементы извлекаются из очереди в порядке, начиная от наибольшего до наименьшего, что делает `priority_queue` идеальным инструментом для ситуаций, когда необходимо регулярно получать доступ к "наиболее значимому" элементу.

Реализация

`priority_queue` реализуется с помощью двоичной кучи, что обеспечивает эффективное извлечение и добавление элементов. Под капотом, `priority_queue` обычно использует `std::vector` для хранения элементов кучи.

Основные операции

1. Вставка элемента (`push`): Элемент добавляется в конец вектора, который поддерживает бинарную кучу, и затем выполняется "просеивание вверх" (heapify-up), чтобы восстановить свойства кучи. Время выполнения этой операции — \(O(\log n)\), где \(n\) — количество элементов в очереди.

2. Удаление наивысшего приоритета элемента (`pop`): Элемент с наивысшим приоритетом (вершина кучи) удаляется. Элемент, который был в конце вектора, перемещается на место удаленного элемента (вершину кучи), после чего выполняется "просеивание вниз" (heapify-down), чтобы восстановить свойства кучи. Эта операция также выполняется за \(O(\log n)\).

3. Доступ к элементу с наивысшим приоритетом (`top`): Возвращает элемент с наивысшим приоритетом (вершину кучи) без его удаления. Это операция выполняется за \(O(1)\).

4. Проверка на пустоту (`empty`) и получение размера (`size`): Эти операции проверяют, пуста ли очередь и возвращают количество элементов в очереди соответственно. Обе операции выполняются за \(O(1)\).

```cpp
#include <iostream>
#include <queue>

int main() {
    std::priority_queue<int> pq;

    // Добавление элементов
    pq.push(10);
    pq.push(5);
    pq.push(20);

    // Вывод элементов от наибольшего к наименьшему
    while (!pq.empty()) {
        std::cout << pq.top() << std::endl; // Печатает наивысший элемент
        pq.pop(); // Удаляет наивысший элемент
    }

    return 0;
}
```

Этот пример демонстрирует, как элементы вставляются в `priority_queue` и извлекаются в порядке убывания их значений.

`priority_queue` — это мощный инструмент для управления данными, когда доступ к наиболее значимым элементам должен быть быстрым и эффективным. Благодаря своей реализации на основе двоичной кучи, `priority_queue` обеспечивает быстрые операции вставки и удаления, что делает его идеальным выбором для многих задач, включая симуляции, планировщики задач, и алгоритмы, требующие частого доступа к элементам с наивысшим приоритетом.
